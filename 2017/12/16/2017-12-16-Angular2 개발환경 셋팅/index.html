Angular2 에 대해서 간단히 소개

구글에서 개발하고 오픈소스로 제공하는 웹 프론트엔드 개발 프레임워크로, SPA개발에 특히 강점이 있는 특징이 있습니다. 컴포넌트 기반의 개발이 핵심이며, 데이터바인딩과 다양한 템플릿 문법을 통해 편리하게 개발을 할 수 있습니다. 더 자세한 정보는 공식문서인 https://angular.io/guide/architecture 이곳을 참고하시면 됩니다.

본격적으로 개발환경을 구축해 보기

Angular에는 Angular Cli라는 커맨드라인으로 편리하게 개발환경을 구축할 수 있는 인터페이스가 존재합니다. 이것을 활용해 보죠.

node가 설치되어 있지 않다면, 먼저 node를 설치해주세요.

npm install -g @angular/cli

이렇게 간편하게 angular/cli를 설치할 수 있습니다. 그런데, 혹 맥 또는 리눅스 환경의 경우 설치 중간에 permission 관련 에러가 생기는 경우가 있는데,

https://docs.npmjs.com/getting-started/fixing-npm-permissions

여기에 올라와 있는 영상을 참고해서 해결해봅시다. 요약하면, npm에서 글로벌 설치하는 경우, 글로벌 설치 디렉터리를 따로 셋팅하고 해당 디렉터리 권한을 설정하는 것입니다.

이렇게 Angular Cli가 잘 설치되었으면, ng 커맨드라인 명령어를 사용할 수 있게 됩니다. 이제 다음과 같이 프로젝트를 생성할 수 있습니다.

ng new 프로젝트 이름(디렉터리 이름) --style css --prefix prefix이름(커스텀 선택자 공통 이름)

다른건 차치하고, prefix에 대해 궁금증이 생길지도 모르겠습니다. 사실 큰 의미는 없고, <dany-product> 이렇게 prefix-component이름 으로 커스텀 선택자가 만들어 지게 됩니다. 개발하다보면 자연스럽게 알게 되니 크게 신경쓸 필요는 없습니다.

이걸로 벌써 Project 생성과 개발환경 셋팅이 끝났습니디! 굉장히 간단하죠?

이제, 자주 사용하는 명령어 / 옵션을 살펴보겠습니다. 이 이하는, 저도 추가로 알게 되는 대로 계속 추가 정리할 예정입니다.

Angular Cli 커맨드 라인 명령어

1.앵귤러 서버 구동

ng server

2.컴포넌트 생성

ng g c (디렉터리/)컴포넌트 이름

3.인터페이스 생성

ng g interface (디렉터리/)인터페이스이름

4.모듈 생성

ng g module (디렉터리/)모듈 이름

실행 옵션

  1. –export
  • ex)

ng g c core/navigation --export

컴포넌트 생성시, –export옵션을 주면 모듈에 자동으로 export가 들어간다.

  1. –is
  • ex)

ng g c banner --is

컴포넌트 생성시, –is 옵션을 주면 css파일이 따로 생성되지 않는다. 배너같이 많은 css가 필요 없는 경우 사용하면 조금 더 편리하게 개발할 수 있다.
