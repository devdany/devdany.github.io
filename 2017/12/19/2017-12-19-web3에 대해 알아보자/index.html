  • dany
  • 2017/12/19
  • Ethereum, Web3
Category: Categories: BlockChain

Web3란?

Web3는 이더리움 노드와 자바스크립트 언어로 통신하기 위한 api입니다. 중요한 것은!! 이 역시 json-rpc로 내부적으로는 동작한다는 것을 잊지 말아야 합니다. 즉, 이전 포스팅에 있었던 자바로 geth 와의 json-rpc 통신 규약에 맞춰 짰던 클라이언트 api와 기본적으로는 다르지 않습니다.(물론 수준은 차이가 많이 나겠습니다만) 결론적으로, 우리는 geth와 통신하기 위해서 json 형태로 데이터를 주고받기 위한 api를 일일이 짤 필요 없이 Web3를 사용하면 아주 심플하게 개발 할 수 있는 것입니다.

Web3 설치하기

저는 어떤 api를 사용하던, 프레임워크를 사용하던 우선 해당 doc를 읽어보는 것이 중요하다고 생각합니다.

https://web3js.readthedocs.io/en/1.0/

바로 여기에 모든게 적혀있습니다. 제가 포스팅에서 소개드리겠지만, 결국 개발을 하시면 위 주소를 북마크에 넣게 되실겁니다. 제가 포스팅 하는 내용 역시 당연히 모두 저기서 따오는 거에요~

일단 Web3를 설치해 봅시다!

npm install web3

엄청 쉽죠? 다른 루트로도 물론 설치할 수 있습니다. 다른 내용은 역시 위 문서에서 참고하시길 바랍니다.

이제 web3가 제공하는 api는 너무나 많기때문에, 자세한 사용 방법은 일일이 여기서 다 적을 수는 없고, 위 공식 doc를 참고하시기 바랍니다.

web3에서 제공하는 api는 많은 경우에 비동기처리, 그중에서도 현재 공식 제공하는 1.0 버전에서는 Promise를 사용한 비동기 처리를 강제하는 경우가 많습니다. 이 역시 마찬가지입니다.

이번엔 이렇게 간단하게 web3를 사용하는 방법에 대해 알아보았습니다.

web3는 개인적으로 정말 편리하고 쉬운 api로, 다른 언어로 이더리움 클라이언트로 만들 수 없는 것은 아니지만, 굳이 javascript를 외면할 이유가 있는 것이 아니라면 javascript와 web3를 활용하는 편이 낫다라는 개인적인 견해입니다. (솔직히 저도 java로 짜봤지만, 굳이 일을 한번 더할 필요가 있을까요?)
