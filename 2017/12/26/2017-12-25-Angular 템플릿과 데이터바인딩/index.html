Angular 템플릿과 데이터 바인딩

  • dany
  • 2017/12/26
  • Angular
Category: Categories: Front-End

템플릿이란?

템플릿은, Html과 앵귤러가 제공하는 템플릿 문법을 활용여 컴포넌트 단위로 뷰를 정의할 수 있도록 해줍니다.

앵귤러에서는, 컴포넌트 클래스로 뷰와 모델을 정의하고, 데이터바인딩을 통해 이 분리된 두 요소가 유기적으로 동작할 수 있도록 합니다.

데이터바인딩?

Angular 단방향 데이터 바인딩과 양방향 데이터 바인딩을 지원합니다. 아래의 7가지 방법을 제공하고 있습니다. 그중 주로 사용하는 것들을 알아보겠습니다.

데이터 바인딩 데이터의 흐름 문법
인터폴레이션 컴포넌트 -> 클래스
프로퍼티 바인딩 컴포넌트 -> 클래스 [property]=”expression”
어트리뷰트 바인딩 컴포넌트 -> 클래스 [attr.attribute-name]=”expression”
클래스 바인딩 컴포넌트 -> 클래스 [class.class-name]=”expression”
스타일 바인딩 컴포넌트 -> 클래스 [style.style-name]=”expression”
이벤트 바인딩 컴포넌트 <- 클래스 (event)=”statement”
양방향 데이터 바인딩 컴포넌트 <-> 클래스 [(NgModel)]=”variable”

인터폴레이션

형태로 데이터를 바인딩 할 수 있습니다. 사실, Angular cli로 처음 프로젝트를 생성하게 되면, 기본적으로 이 방법이 사용되고 있습니다.

HelloComponent의 type 프로퍼티가 템플릿 HTML요소의 Dom 프로퍼티에 바인딩 되고 있는 것을 볼 수 있습니다.

이벤트 바인딩

(event)="status" 형태로 이벤트를 바인딩 할 수 있습니다. 이 이벤트는, 컴포넌트에서 클래스로 바인딩됩니다.

click이라는 이벤트에 컴포넌트 클래스에서 선언한 click()메서드를 바인딩하였습니다.

양방향 데이터 바인딩

양방향 데이터 바인딩은, 뷰와 모델이 상호간에 데이터 상태변화를 인지하여 바인딩하는 것을 말합니다.

즉, 뷰가 변하면 모델도 변하고, 모델이 변하면 뷰도 변하는 것이죠.

이때, input에 값을 입력하게 되면 그 value가 변하면서 name 프로퍼티를 변화시키고, 그 값은 인터폴레이션으로 바인딩됨을 확인할 수 있습니다.
