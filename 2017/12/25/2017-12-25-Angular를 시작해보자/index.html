Angular를 시작해보자

  • dany
  • 2017/12/25
  • Angular
Category: Categories: Front-End

프로젝트 생성

Angular Cli를 전역설치하게되면, 터미널 명령어로 간단하게 앵귤러 프로젝트를 생성할 수 있습니다.

ng new 프로젝트이름( ex - helloAngular )

프로젝트 실행

앵귤러는 다음과 같은 명령어를 입력받으면, webpack을 사용하여 소스코드를 번들링하고, 개발용 내장서버를 사용하여 프로젝트를 실행합니다.

ng serve

프로젝트 구성

앵귤러는 각종 설정파일 및 리소스를 제하면, 다음과 같은 프로젝트 구성을 가지고 있습니다.

app

앵귤러는 기본적으로 최소한 하나의 모듈과 하나의 컴포넌트를 필요로 하는데, Angular Cli를 사용하며 프로젝트를 생성하게 되면 이를 자동적으로 생성해 줍니다.

이 app 디렉터리 안의 app module과 app component가 바로 그것인데, 이것이 루트 모듈/컴포넌트가 됩니다. 루트모듈은 main.ts에 의해 부트스트랩되고, 루트컴포넌트는 루트모듈에 의해 부트스트랩됩니다.

컴포넌트 구성

현재 나오는 앵귤러 역시 React, Vue 등과 같이 컴포넌트 기반의 웹 개발 프레임워크입니다. 따라서, 역시 뭐니뭐니 해도 핵심은 컴포넌트일 것입니다. 이 앵귤러 컴포먼트를 살펴보겠습니다.

컴포넌트는 import영역, 데코레이터 영역, 클래스 영역으로 나눌 수 있습니다.

import 영역

해당 컴포넌트가 필요로 하는 외부 모듈 임포트를 선언합니다.

데코레이터 영역

인자로 전달할 컴포넌트 생성에 필요한 메타데이터(셀렉터, 템플릿, 스타일) 객체를 선언합니다.

클래스 영역

컴포넌트의 뷰를 관리하기 위한 로직이 들어갑니다.

컴포넌트 생성

그럼 컴포넌트를 생성해 보겠습니다.

ng g c hello

그림처럼 컴포넌트가 생성된것을 볼 수 있습니다.

hello컴포넌트 템플릿 파일(hello.component.html)에 다음처럼 입력해봅시다.

css적용을 테스트 해보기 위해 css파일도 수정해봅니다

루트모듈(app module)이 helloComponent를 포함하고 있음을 확인합니다.

루트컴포넌트의 템플릿에 helloComponent를 포함시킵니다.

아래처럼 컴포넌트가 제대로 나왔으면 성공입니다!
