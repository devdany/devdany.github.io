RxJs 번역 및 정리 시작

RxJS는 자바스크립트 비동기 리액티브 프로그래밍에서 좋은 퍼포먼스를 보이며, Angular에서 이를 잘 활용하고 있다고 생각합니다. 그동안은 얕게 사용해왔지만, 한번 공식 문서를 직접(구글 번역기가) 번역하고 정리해보려고 합니다. 또한 다른언어(RxJava 등)에서도 마찬가지이기 때문에, 한번 정리해두는 것이 큰 도움이 될거라 생각합니다. 제가 뭐 영어를 특출나게 하지도 않고 RxJS를 엄청 잘 아는것도 아니지만, 따로 와닿는 정리본이 없으니 어쩌겠어요 ㅋㅋ 직접해야지.. 그럼 시작해보겠습니다! 다 끝낼 수 있길!

Introduction

RxJS는 Observable sequence를 사용하여 비동기 및 이벤트 기반 프로그램을 작성하기위한 라이브러리입니다. 코어타입인 Observable과 그 외 타입(Observer, Scheduler, Subject) 그리고 배열과 여러 함수들(map, filter, reduce, every)등에 영향을 받는 연산자를 사용하여 비동기 이벤트를 콜렉션으로 처리할 수 있습니다.

RxJS의 핵심 개념은 다음과 같습니다.

  • Observable : 호출 가능한 미래의 value나 event 콜렉션을 나타냅니다.
  • Observer : Observable에 의해 전달 된 value를 listen하는 콜백 콜렉션입니다.
  • Subscription : Observable의 실행을 나타내며 주로 실행 취소에 유용합니다.
  • Operators : map, filter, concat, flatMap 등과 같은 연산을 사용하여 컬렉션을 다루는 함수형 프로그래밍 스타일을 가능하게하는 순수한 함수입니다.
  • Subject : EventEmitter와 동등하며 여러 Observers에 값 또는 이벤트를 멀티 캐스팅하는 유일한 방법입니다.
  • Schedulers : 동시실행을 제어하기 위 중앙화된 Dispatcher로, 연산이 e.g. setTimeout or requestAnimationFrame or others 등에서 언제 발생하는지를 구조화할 수 있도록 합니다.

Pull vs Push

Pull과 Push는 데이터 생산자와 데이터 소비자가 어떻게 통신하는지를 나타내는 두가지 다른 프로토콜입니다.

Pull:

Pull시스템에서는 데이터 소비자가 데이터 생산자로부터 데이터를 받을 시기를 결정합니다. 데이터 생산자는 언제 데이터가 전달되는지 알지 못합니다. 모든 자바스크립트 함수는 Pull시스템입니다.

Push:

Push시스템에서는 데이터 생산자가 데이터 소비자에게 언제 데이터를 보낼것인지를 결정합니다. 데이터 소비자는 언제 데이터를 받게될지 알지 못합니다. 오늘날 자바스크립트에서는, Promise가 Push 시스템의 가장 일반적인 형태입니다.

RxJs는 자바스크립트의 새로운 Push시스템입니다. Observable은 Multiple데이터의 생산자이고, 이 데이터를 데이터 소비자인 Observer에게 Push하죠.

Observables as generalizations of functions

많은 주장들과 달리, Observable은 EventEmitter같은것도 아니고, Multiple data를 위한 Promise같은것도 아닙니다. Observable은 어떤 경우에는(RxJS Subject를 사용해서 멀티캐스팅 되었을때) EventEmitter처럼 동작할 수 있습니다. 그러나 일반적으로는 EventEmitter처럼 작동하지 않습니다.

subscribe Observable

Observable은 이렇게 subscribe될 수 있습니다.

observable.subscribe와, Rx.Observable.create(function subscribe(observer){}) 에서 인자로 넘기는 함수 subscribe의 이름이 같은것은 우연의 일치가 아닙니다. 라이브러리상에서는 분명 그 둘은 다르지만, 실제 쓰임에서는 개념적으로 동일하다고 볼 수 있습니다.

이것은 하나의 Observable에 대해 복수의 Observer가 어떻게 subscribe call을 공유하지 않는지 보여줍니다.

observable.subscribe를 콜할때, Observable.create의 인자로 들어갔던 subscribe function은 해당 Observer에 대해 실행됩니다.

그래서 observable.subscribe는 해당 옵저버에 대해서만 독립적으로 실행된다고 볼 수 있습니다.

이것은 addEventListener / removeEventListener와 같은 이벤트 핸들러 API와 크게 다릅니다.

observable.subscribe를 사용하면, 해당 Observer는 Observable에 리스너로 등록되지는 않습니다.

Observable은 붙어있는 Observer의 리스트 조차 갖고있지 않습니다.

subscribe calld은 단순히 Observable 실행을 시작하고, 그 결과값이나 이벤트를 Observer에게 건네주기 위한 방법일 뿐입니다.

Executing Observable

Observable.create (function subscribe (observer) {…}) 내의 코드는 subscribe하는 각 Observer에 대해서만 발생하는 lazy한 실행인 “Observable 실행”을 나타냅니다.

그 실행은 동기적 또는 비동기적으로 실행의 여러 결과값을 생성해냅니다.

Observable Execution이 제공 할 수있는 값은 세 가지 유형이 있습니다.

  • Next : number, String, Object등을 전달
  • Error : JavaScript Exception을 전달
  • Complete: value를 전달하지 않음.

Next 알림이 가장 중요하며 일반적인 형태인데, 이는 Observer에게 데이터를 전달함을 나타냅니다. Error와 Complete는 Observable실행중 단 한번만 일어납니다.

Disposing Observable Executions

Observable Execution은 무한적이고 Observer는 한정 시간에 실행을 중단하려는 것이 일반적이므로 실행을 취소하기위한 API가 필요합니다.

각 실행은 하나의 옵저버 전용이기 때문에, 옵저버가 값을 받으면 실행을 중지하는 방법이 있어야 계산 능력이나 메모리 리소스가 낭비되지 않습니다.

당신이 subscribe할때, subscription을 리턴받는데 그 객체의 unsubscription을 콜하면 됩니다.
