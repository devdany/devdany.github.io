Angular HttpClient

Angular 4.3부터는 AngularHttp라는 클래스를 사용할 수 있습니다. 이 클래스는 기존의 것보다 더 발전된 Request API와 인터셉터를 제공하고 있습니다.

HttpClient는 Observable을 반환합니다. 즉, RxJS의 Observable객체로 이루어져 있습니다.

사용법 또한 간단합니다. HttpClientModule을 Import하는 것만으로 사용할 수 있습니다.

이렇게 모듈에 임포트 하는것으로 충분합니다. 이제, 컴포넌트나 서비스에 HttpClient를 주입할 수 있습니다.

Get Request

간단한 Get Request 요청 예제입니다.

생성자에서 HttpClient를 간단하게 주입받고, test() 에서와 같이 요청을 보낼 수 있습니다. 그리고 그 요청은 Observable 객체를 리턴하므로, subscribe를 해주어야 합니다.

그런데 이때 주의해야 할 점이 하나 있습니다. 예제와 같이 보겠습니다.

이 경우 서버에서 array를 리턴하더라도 에러가 납니다. 왜냐하면 이 Request메서드는 제네릭을 사용하기때문에, 타입을 반드시 명시해줘야 합니다.

이렇게 말이죠.

Response Type

또, 서버로부터 받을 데이터가 Json이 아닌 text 등의 데이터일 경우에는 Response Type을 정해줘야 합니다.

이런 형태입니다. 이때에는 제네릭으로 타입을 지정해줄 필요가 없습니다.

HttpParams

다음은, 파라미터를 보내는 방법에 대해 정리해보겠습니다.

물론, url을 일반적인 Get방식의 파라미터를 추가하는 방법처럼 ?key=value 를 나열해도 상관없습니다만, HttpParams를 사용해서 Param을 좀더 깔끔하게 정리할 수 있습니다.

Post Request

post요청에는 두번째 파라미터로 body에 들어갈 데이터를 줘야 합니다. 간단한 회원가입 예제를 생각해 볼까요?

이렇게 보낼 유저를 오브젝트로 정의하고, 두번쨰 파라미터에 정의되는 body에 담아주면 편합니다.

이렇게 Angular에서 Http요청을 보내는 방법을 간단히 알아봤습니다. 다음에는, Angular에서 제공하는 Form 템플릿 활용에 대해 정리해보겠습니다.
