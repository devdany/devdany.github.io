SPA와 Routing

  • dany
  • 2018/01/04
  • Angular
Category: Categories: Front-End

SPA

SinglePageApplication로, 모던 웹의 패러다임으로 떠오르고 있는 개념입니다. 포인트는 아래와 같습니다.

  1. 단일페이지로 구성된다.
  2. 배포가 간단하다.
  3. 좋은 사용자 경험을 제공할 수 있다.

SPA는 웹 애플리케이션을 구동할때, 처음 한번에 모든 정적 리소스를 서버로부터 가져와 렌더링합니다. 그리고 다시 서버로 리퀘스트를 보내고 페이지를 새로고침하지 않고 웹 애플리케이션을 사용하도록 합니다.

Angular Routing

Angular는 SPA를 위한 프론트 사이드 Router를 제공합니다. Angular는 사용자의 요청 URL을 해석해서, 선언적 방식으로 매핑된 컴포넌트를 연결해줍니다.

Angular Router 사용

1.선언

Routes 인터페이스의 배열을 사용하여 요청 URL과 컴포넌트로 만들어진 라우트를 선언한다.

2.등록

RouterModule.forRoot 또는 RouterModule.forChild를 호출하여 라우트 구성이 포함된 모듈을 생성하고, 모듈에 등록한다.

3.뷰의 렌더링 위치 지정

라우터가 컴포넌트를 렌더링하여 뷰를 표시할 영역인 을 구현하는 RouterOutlet 디렉티브를 선언하여 컴포넌트 뷰가 렌더링될 위치를 지정한다. RouterOutlet 디렉티브는 컴포넌트에 등록한다.

4.네비게이션 작성

RouterLink디렉티브를 사용해서 네비게이션 링크를 작성한다.

그럼, 예제를 작성해보겠습니다. 적당하게, first, second, third, last 컴포넌트 4개를 만들겠습니다. 그리고 잘못된 요청에 대비한 notFound 컴포넌트도 하나 만들겠습니다.

ng g c first

ng g c second

ng g c third

ng g c last

ng g c notFound

그리고 루트모듈에 다음과 Routes 인터페이스의 배열로, URL과 각 컴포넌트를 매핑합니다.

그리고 이렇게 구성된 Routes를 Router모듈에 등록하고, 이 모듈을 루트모듈에 등록합니다.

다음 라우터에 의해 컴포넌트가 렌더릴될 영역을 <router-outlet> 로 지정하고, 네비게이션을 만듭니다. 실행해보면, 정상적으로 각 컴포넌트가 렌더링되는 것을 볼 수 있습니다.

RouterLinkActive

RouterLinkActive 디렉티브를 사용한 요소는, 현재 라우트 상태가 포함된 요소에 css클래스가 적용됩니다. 한번 적용시켜 보겠습니다.

이렇게 하면, /first 에서는 first메뉴만 큰폰트에 빨간 글씨로 나타나는 것을 볼 수 있습니다.

routerLink Parameter

링크시, 파라미터를 사용해서 다른 컴포넌트로 데이터를 전달할 수도 있습니다.

먼저, 선언시 파라미터를 이름을 지정합니다.

{path: 'last/:num', component: LastComponent},

이렇게 하면, last/10 이라고 왔을때, num이라는 파라미터 name으로 10이라는 value를 받습니다.

그리고, 해당 컴포넌트에서 ActivatedRoute를 생성자에 주입받습니다.

즉, 이 오브젝트를 subscribe 해서 인자로 전달되는 파라미터를 얻을 수 있습니다.
